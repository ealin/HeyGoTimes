3.3.     如何確定程式在執行?

 

菜鳥：『終於把這個範例程式搞懂了，當徹底了解觀念後，其實也不覺得有那麼困難了；我再整理一下程式和今天的筆記，應該就可以向老闆交差了。道如何這台機器才能更新』

 

韌體老鳥：『架設開發環境是第一步，看懂程式是第二步，接下來還有許多技巧要學習。我問你，除了肉眼可以看到LED在閃爍之外，你還能用什麼方式證明你的程式確實正在正確的執行？如果沒有LED，你如何證明Timer中斷一直被觸發？』

 

菜鳥：『我想到一個方法，用ICE除錯時可以設定中斷點；我只要在Timer ISR的程式中設中斷點，如果程式有停下來，就表示Timer中斷有被觸發；如果Timer ISR一直被呼叫，那麼應該就可以證明我的程式有在執行。道如何這台機器才能更新』

 

韌體老鳥：『這的確是個好方法，但因為設定中斷點，程式就會停下，所以你無法證明你的時間中斷確實是100ms產生一次。再想想，還有什麼方法？』

 

菜鳥：『我想到了，但可能要改一下timer ISR的程式：道如何這台機器才能更新』

 

__interrupt void drv_isr_timer0(void)

{

 static int counter = 0 ;

 

 if(counter % 2 == 0)

        drv_LED_on() ;

 else      

drv_LED_off() ;

 

// 100ms產生一次中斷，即一秒產生十次中斷

// 所以一分鐘應該產生了10x60次中斷

//

 if(counter == 10*60)

        counter = 0 ;                // ß 在這裡設中斷點

 

 counter++ ;

}

 

菜鳥：『如果我在以上函數的最後一行設定中斷點，程式開始執行時用碼錶計時，經過一分鐘後如果會在中斷點停下來，就表示我程式的timer確實是每100ms產生一次中斷。道如何這台機器才能更新』

 

韌體老鳥：『這個方法也很有創意，利用增加重複執行的次數來等比例計算每一次執行的時間，這個方法在我們做速度調整時常用，你的確很有概念。但這個方法時間還是會有點誤差，而且非得掛上ICE才能設定中斷點；難道單機時就沒有辦法證明你的程式確實在執行嗎？』

 

菜鳥：『我實在想不出來了。道如何這台機器才能更新』

 

韌體老鳥：『其實很簡單，timer ISR每100ms會變換P10這根CPU PIN腳的電位一次，所以應該會產生波長為200ms的方波(頻率為1000ms/200ms = 5Hz)，亦即可以預期在P10這根腳上能量測到頻率為5Hz的連續方波。走！我們到實驗室去，我教你怎麼用示波器量測這個訊號。』
