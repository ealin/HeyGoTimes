附錄B: Callback Function

 

如果程式A中使用了某個function B，我們會說程式A呼叫(call)了function B；如果某個系統模組的API要求你給傳入一個函數指標(function pointer)，而這個函數指標指到一個實際的函數 (多半這個函數是你自己寫的)；然後系統模組會在適當的時間呼叫此函數，則此函數就是所謂的 callback function。因為這個函數是被『callback』了(讀者可以理解為“回call”)。

 

這樣的技巧廣泛的使用在系統與驅動程式中；以系統處理鍵盤事件為例，當某個key被按下時，keyboard driver會辨認哪一個key被按下了，並且把這個事件送給OS處理，通常OS在經過某種轉換後也會把這個key的訊息存在某個queue中，最後才交予應用程式處理。

 

我們假設一種狀況，user連續按下了A、B、C、D與ENTER共五個key，應用程式處理key A需要的時間較長，則其他四個key事件會被存在queue中，暫時無法被處理。若此時使用者等得不耐煩了，按了ESC想要跳出這個功能，應用程式該如何對應？

 

在一般的設計架構中，整個流程應用程式完全無法插手，而OS也無法定義ESC key的意義為何；所以在上述狀況中，ESC key也會被存在queue中，要等到前五個key都處理完後，應用程式才知道原來ESC key已經被按下了。Callback function可以解決這個問題，讓應用程式設計更有彈性；簡單的說，應用程式可以先對keyboard driver或OS註冊處理ESC key的function pointer，則每當有key事件發生時，keyboard driver或OS除了執行原來的流程外，在某個階段就會去呼叫應用程式註冊的function，所以ESC key就可以提前被處理。

 

請參考以下程式範例：

 

// define.h

// 定義key callback function的prototype。

//

// 傳入一個int表示key的編號

//

typedef void (* key_callback) (int);

 

 

 

// drv_key.c

// keyboard driver

//

#include <define.h>

 

// Global Variable

//

key_callback sys_key_callkack = 0 ;

 

 

// Function: key_register_callback

//    用途：讓應用程式可以註冊keyboard callback

//     參數：傳入新的callback function的指標

//

void key_register_callback(key_callback new_callback)

{

 sys_key_callkack = new_callback ;

 

}

 

// Function: key_process_key_event

//    處理key event, 中間會去呼叫sys_key_callback()

//

void key_process_key_event(int key)

{

 ... 處理kev event

 ...

 

 if(sys_key_callback != 0)  // 已經註冊過callback function

        sys_key_callback(key) ;   // 呼叫應用程式的callback function

 

 ... 繼續處理key event

 ...

 

}

 

 

// application.c

// 應用程式

//

// Function: app_key_callback

//    keyboard的callback function, 當某個key事件發生時

//     keyboard driver會呼叫這個函數

//

void app_key_callback(int key)

{

 if(key == KEY_ESC)

 {

        // 處理ESC KEY

 }

 

}

 

// Function: app_main

//     應用程式的主程式, 必須儘早註冊key callback function

//

void app_main(void)

{

...

...

 //註冊key callback function

 //

 key_register_callback(app_key_callback) ;

...

...

}
