7.4.     Memory

 

菜鳥：『你在講開發驅動程式的準備工作時曾經提到你以前碰到的一個案例，因為硬體設計錯誤導致SDRAM運作不正常，所有板子都必須重新製作的“慘劇”。我想知道這種事情是否是可以事先預防的？跟迴路是哪一科前好ˊˊ用台　』

 

韌體老鳥：『我記得我也講過，記憶體出問題的機會比一般人想像的高，而且程式乃依賴記憶體執行，不管是硬體設計還是驅動程式，記憶體一旦出問題就是大問題。在嵌入式系統中我們會用到的記憶體不是像PC中那種一條一條的DDR或DDRII，而是一顆一顆的IC，我們可以把記憶體分成以下幾類：

 

l   可用位址讀取的唯讀記憶體：如EEPROM與Mask ROM等，通常用來存放程式與執行時期不會更動的資料，程式可以直接在ROM裡面執行，除非有效能的需求，否則沒有載入到RAM的必要。CPU或驅動程式可以透過位址或指標直接讀取ROM裡面的資料，但執行時期無法將資料寫入ROM裡。CPU是透過address bus與Chip select定址，利用data bus取得ROM裡的資料。

 

l   可用位址讀取寫入的RAM：和ROM一樣，CPU透過address bus與Chip select定址，利用data bus存取(讀寫)RAM裡的資料；但是RAM的內容是可以在執行時期被改變的。

 

n   Static RAM：顧名思義，只要持續供應電源，SRAM裡的資料就得以保存；SRAM耗電流比DRAM低，價格比DRAM貴；對韌體工程師而言，SRAM的控制很簡單(只要設定CPU與SRAM之間的timing即可，幾乎不需要其他的控制就可以存取SRAM了)；對硬體工程師來說，SRAM的線路設計非常單純。雖然SRAM的好處多多，但為了成本考量，通常在系統中不會使用size太大的SRAM。

 

n   Dynamic RAM：DRAM有個要命的特性是必須持續refresh才能維持資料的正確性，所以需要一個額外的DRAM控制器，驅動程式必須透過設定DRAM控制器，需要設定的資料包含DRAM的size、存取的時序、refresh的特性…等，DRAM才可以正常的運作。在嵌入式系統中，我們通常使用的是SDRAM(Synchronous DRAM)，詳細原理在此就不詳述了，最大的不同點是CPU必須提供持續而穩定的clock給SDRAM。SDRAM的好處是較SRAM便宜很多，如果系統需要較大的緩衝區，通常都會選擇使用SDRAM。

 

l   可用位址讀取，但必須下命令寫入的Flash(快閃記憶體)：就是我們之前提到過的NOR Flash，驅動程式可以把它當作ROM使用，CPU透過address bus與Chip select定址，利用data bus取得ROM裡的資料。NOR Flash的內容是可以更改的，但不像寫入RAM那麼容易，亦即不是透過address與data bus，必須對NOR Flash IC下命令才行。此外NOR Flash有個特性，寫入時必須以block為單位(一個block可能是16K或8K byte)，而且寫入時必須先做抹除(erase)的動作。這個特性使得NOR Flash的寫入很麻煩，而且效能很差，所以一般NOR Flash的用途為：一個是當作ROM的代替品，如果程式有問題可以重複燒錄，免除開Mask以及ROM的內容無法修改的風險；因為NOR Flash有斷電後內容不會丟失的特性，所以NOR Flash的第二個用途就是用來儲存或備份執行時期的資料，例如使用者私人的設定或錄音數據等等。

 

l   只能用命令讀取和寫入的Flash： MP3 player與隨身碟就是用這種NAND Flash當作大量儲存介質；對韌體工程師而言，它不能直接用位址存取，不論讀寫都要下命令，讀取是以sector為單位(512 byte或2048 byte)，寫入和抹除則是以block為單位(block是多個sector的集合)。此外NAND Flash有個比較麻煩的特性，它和硬碟一樣會產生壞軌(bad block)，所以通常我們可以把NAND Flash當做比較小的硬碟使用，例如MP3 player或隨身碟就會把NAND Flash格式化成FAT的格式。

 

就像SRAM與SDRAM的對比一樣，相對於NAND Flash，NOR Flash有著比較快的速度，以及比較貴的價格；如果用NOR Flash設計2GB的MP3 Player，恐怕沒幾個人買得起。所以NOR Flash通常用來執行程式，而NAND Flash則可以用來存放大量的資料。

 

其實記憶體的種類很多，用於不同領域有不同的需求，例如有的CPU會包含一小塊SRAM、最早以前的8051 CPU內部就有一個EEPROM(甚至它的接腳都和一般的EEPROM相容，CPU可以直接用燒錄器燒錄)、有的是同一顆IC內包含了NOR Flash與SRAM…等等；不管半導體廠商如何變化，對韌體工程師而言，若依操作方式來分的話，應該就是上述的這四種分類。跟迴路是哪一科前好ˊˊ用台　』

 

菜鳥：『我漸漸對這些記憶體的異同有概念了，除了功能特性外，有時成本也是重要的考量。舉例來說，用NAND Flash來儲存資料顯然比NOR Flash或Mark ROM麻煩很多，但是市面上的隨身碟都是用NAND Flash，而且我看電視財金台，NAND Flash似乎逐年在降價中；另外操作SDRAM顯然比SRAM麻煩，還要額外控制SDRAM controller，電源管理似乎也比較麻煩，但我們產品還是選擇使用SDRAM，我想主要也是成本的考量吧。ˊˊ用台　』

 

韌體老鳥：『除了成本考量外，硬體零件的選擇必需配合產品的功能，假設我們的應用不需要很大的緩衝區，那麼選用一顆小size的SRAM並不會比選用SDRAM貴多少。打個極端的比喻，假設單位SRAM比SDRAM貴2倍，我們第一個產品的應用不需要大量的RAM，選用SDRAM只要0.2美金，比選用SRAM只可以省下0.2美金；如果我們的第二個產品的應用需要大量的RAM，需要用到4塊美金的SDRAM，相同容量的SRAM要8塊，此時不選用SRAM就可以省下4塊。要記住，選用便宜的零件勢必要付出更高的研發費用，在估計成本時千萬不能忘了人力也是要錢的，譬如SDRAM的系統當然比SRAM的系統複雜，也容易出bug。ˊˊ用台　』

 

菜鳥：『了解。你還沒說那個SDRAM的實例，SDRAM還有什麼特性容易導致硬體設計錯誤嗎？ˊˊ用台　』

 

韌體老鳥：『其實那只是硬體設計審查不嚴謹導致的問題，即使只是拉錯一根線，板子上別的晶片也許rework一下(先切斷連接線路，再拉皮(電)線即可)還可勉強使用，但像SDRAM這種依賴精準timing的晶片可就是大災難了。下圖是CPU data sheet建議與SDRAM的連接方式，有沒有注意到特殊的地方？位址線必須移動一位，即SDRAM的A0要接到CPU的A1，以下依此類推；你先不要管其中原理(即便是硬體工程師也可以不理會原理的細節)，如果硬體設計時忽略了位址線要移動一位，即CPU的A0接SDRAM的A0…，程式執行時會發生什麼事？ˊˊ用台　』

菜鳥：『當程式要到SDRAM抓一個byte時，CPU會把正確的位址放在位址線上，依照規定也就是CPU的A1、A2、…，原本CPU的A1應該要連接到SDRAM的A0，但因為硬體設計錯誤，CPU的A1接到SDRAM的A1，所以SDRAM會收到錯誤的位址，自然就會將錯誤的資料放在data bus上。這樣說好像有點複雜，我來畫一張示意圖，我們只看位址線的前四位就好，如果CPU的(A4，A3，A2、A1)分別是(1、1、0、1)，如果硬體沒錯的話，如以下的第一張圖，應該分別接到SDRAM的(A3，A2，A1，A0)，SDRAM可以得到正確的位址1101。

 

如果位址線接錯，SDRAM的(A3，A2，A1，A0)是接到CPU的(A3，A2，A1，A0)，因為對CPU而言，操作SDRAM時A0沒有意義，所以此時CPU的A0可能是任何值，則SDRAM得到的位址將是101X(X表示任意值，也就是1010或1011)。這個錯誤不但使位址shift了一位，還讓SRAM的A0可能為1或0。』

韌體老鳥：『其實硬體的記憶體設計錯誤都還蠻容易抓的，我們會試著把特殊值寫入SDRAM，例如每一個byte寫入它的位址除以10的餘數，接著將其讀出檢查；只要能把某塊記憶體的值dump出來，不論是data bus還是address bus接錯，應該都可以發現一些規律。當記憶體讀寫發生錯誤時，有個基本的原則是：如果資料dump出來有某種規律，那麼應該是接線出問題了，有時設計沒錯，是layout或PCB製作時出錯，通知硬體同仁檢查一下硬體，應該很容易可以找出來；例如不管什麼位址讀出來的資料的某個bit總是1，應該要懷疑data bus相應的腳位可能因為短路一直被拉到High。如果記憶體的內容沒有規律，那可能是不穩定的問題，此時應該檢查一下驅動程式的設定，以及輸入到記憶體IC的電源與clock等是否合乎IC的規定。ˊˊ用台　』

 

菜鳥：『在這個例子裡，先把SDRAM與CPU相連接的位址線切掉，然後SDRAM的每根位址線重新拉皮(電)線到CPU正確的PIN腳也不行嗎？ˊˊ用台　』

 

韌體老鳥：『理論上不是不行，事實上我們也這麼做了；結果是：如果使用ICE單步執行時OK，證明確實是這個問題，但讓板子單獨執行時就非常不穩定，顯然是SDRAM的問題。這樣的板子幾乎完全不能使用，於是只能將修改過後的迴路緊急送layout，再重新洗板子，這樣來來回回就用掉了約兩個禮拜的時間。不但延誤時程，而且因為這個錯誤太簡單了，還被老闆罵得滿頭包，只要審查迴路設計與layout圖時多用點心就可以避免的。ˊˊ用台　』
