0.6.     讀書計畫

 

專案經理：『如果你還擔心的話，我開一些書單給你，你可以去找一些相關領域的書來看，就當是職前自我訓練吧：一些書單的話幫;1;式息是的狀況通常是』

 

 

l   計算機組織或計算機結構：確定你對計算機系統運作的原理都清楚嗎？把教科書翻出來複習一下吧！

 

l   嵌入式系統概論：市面上相關書籍很多，有的介紹概念，有的就是收集一堆業界相關資料，有的則侷限於某個平台上的實作；對初進入這個領域的人而言當然都有翻閱的價值，多充實一些相關知識總是好的。但筆者以為觀念甚於一切，業界資訊或某種技術的詳細資料隨時上網就可找到，但正確的觀念卻是需要養成的。以下推薦一本經典級的書：

 

Programming Embedded System in C and C++

(中文版：嵌入式系統-使用C/C++書的是好的ˊ)

 

l   嵌入式作業系統：市面上與嵌入式作業系統相關的書也不少，筆者建議可優先從μC/OS II的書開始讀起。業界有許多知名的嵌入式作業系統，如VxWroks、Nucleus、QNX…等，其運作原理都和μC/OS II大同小異，故以下這本書相當具有代表性：入式作業系統　

 

μC/OS II The Real-Time Kernel

(中文版：嵌入式實時操作系統μC/OS II)

 

l   驅動程式：如果要學嵌入式系統驅動程式開發，千萬不要買了本Windows device driver或Linux device driver的書硬K，基本上那對開發嵌入式系統的驅動程式沒什麼幫助，因為這些書大部分的篇幅都在講解Windows或Linux對驅動程式開發的“規定”。筆者推薦一本詳述如何在μC/OS上開發驅動程式的參考書，其中包含了很多實例，讀者可將其當作嵌入式系統驅動程式開發的入門書：

 

Embedded Systems Building Blocks – Complete and Ready-to-use Modules in C.

 

還有一本也是不錯的參考書：

 

Embedded Systems Firmware Demystified.

 

 

l   硬體設計概論：市面上有些從硬體設計出發講述嵌入式系統概論的書，軟體領域的人可以從中習得基本的硬體知識；筆者推薦：

 

Designing Embedded hardware (O’REILLY出版)

(中文版：設計嵌入式硬體)

 

 

嵌入式系統這個領域比較特殊，很不容易找得到相關的教育訓練課程，就算有通常也是概論居多。舉例來說，市面很容易找到PHP+MySQL網站設計或微軟、Linux認證的課程，但肯定找不到諸如“Epson SX5580 CPU[1]開發實務”、“印表機專用軟體設計”這樣的課程；原因無他，就是因為嵌入式系統沒有一定的硬體配置與應用需求。所以嵌入式系統開發團隊安排工程師參加外部教育訓練的機會比較少，通常都得靠團隊內部的經驗傳承、自行搜尋資料、或者請供應商技術支援等。

 

最後還是要再提一次，嵌入式系統開發最重要的是觀念；筆者認為如欲養成正確之嵌入式系統發開發的觀念，上述書單都是值得一讀的重要參考書。

 

 


[1] Epson SX5580是一顆客製化的CPU，專門用在電子字典以及低階PDA產品的開發。
