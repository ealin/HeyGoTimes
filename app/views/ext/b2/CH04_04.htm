作業系統（Operating System，簡稱OS） 是一管理電腦硬體與軟體資源的程式，同時也是電腦系統的核心與基石。OS身負諸如管理與配置記憶空間（Memory space）、決定系統資源供需的優先次序、控制輸入與輸出裝置、操作網路與管理檔案系統等基本事務。OS也提供一個讓使用者與系統互動的操作介面。

 

OS的型態非常多樣，不同機器安裝的OS可從簡單到複雜，可從手機的嵌入式系統到超級電腦的大型操作系統。許多OS製造者對OS的定義也不大一致，例如有些OS整合了圖形使用者介面，而有些OS僅使用文字介面，而將圖形介面視為一種非必要的應用程式。[1]

 

 

嵌入式作業系統

專案經理：『許多嵌入式系統的應用非常單純，這樣的產品幾乎不會用到作業系統的概念，軟體架構就像你之前寫的LED閃爍程式(請複習一下第三章)，充其量就是一個boot程式加上許多函式庫，以及應用程式而已。

 

但現今的電子產品的應用日趨複雜，所以系統中確實需要一個“核心”，負責工作排程(scheduling)、記憶體管理、檔案系統等基本的系統服務，不管產品的應用為何都需要這個核心程式，我們稱這個“系統裡的核心”為嵌入式作業系統。目前市面上可選用的嵌入式作業系統很多(請參考下表，讀者可自行到各個嵌入式作業系統的官方網站瀏覽)，各有各個優點，不同的應用領域偏好使用不同的嵌入式作業系統，例如網路產品大多使用Embedded Linux或VxWorks，因為它們網路功能支援較好，而目前手機產業則偏好使用Nucleus、Symbian OS、PocketPC SmartPhone Edition(Windows Mobile)或μC/OS II。

 

此外網路上open source的嵌入式作業系統很多，其中不乏已廣泛應用在商業電子產品之上者，例如：μC/OS II、Embedded Linux及ECos等。值得一提的是電子產品製造大國 – 日本，自行訂定了嵌入式系統的國家標準 –μITRON，其鉅細靡遺的定義了Embedded OS的行為以及API，而幾乎所有日系的CPU都會搭配一套與μITRON相容的Embedded OS；這代表的意義是：之前在其他CPU平台開發的程式，可以很輕易地移植到其他平台。日本電子產品總是可以標榜低成本高品質，其來有自。植郝呃有的大國作業系統好功能統服務檔案系統等等業系統之前入是來的範圍　養如何我開發了一個簡單的時候多讀完了經有域變數統中況的多吧』

即時系統

菜鳥：『我在學校學過即時系統的概念，Real-Time分為Hard Real-Time與Soft Real-Time，前者最有名的例子就是核電廠控制系統，若使用Windows XP來控制核電廠，則當發生緊急狀況時，什麼時候可以得到正確的反應完全無法預測，所以Windows XP必然不是一個Hard Real-Time系統，因為其反應時間不可預測。而Soft Real-Time系統則是用來處理延遲反應的後果不那麼嚴重的情況。我現在不懂的是，為什麼嵌入式作業系統要稱為RTOS，和Real-Time有什麼必然的關係嗎本書的原因？稱為作業系統到正確的反應我們明外域變數統中況的多吧』

 

專案經理：『這是一個好問題；我們在PC可以忍受Windows把明明很快的硬體設備弄得慢吞吞，最多就是暗幹一聲，但是你可以忍受煞車系統兩秒鐘後才反應嗎？與人身安全有關的系統當然要求即時反應，但就算是電視機、微波爐、電子辭典、手機、甚至電子雞，按了鍵後有時反應很快，有時要兩秒鐘後才會反應，你覺得這種電子產品賣得出去嗎？更不要說反應延遲在GPS、醫療設備、航空電子等領域會帶來什麼後果。

 

不論是什麼應用，使用者就是無法忍受電子產品的反應遲鈍或不可預測；所以嵌入式作業系統有幾個基本的要求：

 

l   可中斷性(Preemptive) – 不管系統目前的狀況是什麼，當緊急事件發生時，OS必須保證相應的程式會馬上被執行；這是即時系統的第一個要求。

 

l   可預測性 – OS中所有函數與服務的執行時間必須是確定的；這是即時系統的第二個要求。

 

l   穩定性與可靠性 – 這是即時系統的第三個要求。

嵌入式系統Task架構實例

專案經理：『那你知道我們系統中有哪些task，交互關係如何嗎？的型太複雜˙麼不同式業系統之前入是來的範圍　養如何我開發了一個簡單的時候多讀完了經有域變數統中況的多吧』

 

菜鳥：『這次開發的產品雖然功能很複雜，但並沒有需要並行的功能，所以基本還用不到多功，也就是說應用程式只需要一個main task處理即可；但因為產品有省電的需求，所以當main task沒有事情做的時候會自行sleep，則RTOS會將控制權交予優先等級較低的idle task，由idle task負責讓CPU進入睡眠模式。

 

當有硬體事件發生時，其ISR會將硬體事件傳入main task的message queue中，此時CPU會從睡眠模式醒來，idle task繼續執行，接著喚醒(wakeup)main task；因為main task的優先等級比較高，所以RTOS自然會將CPU執行權交給main task，則main task可以處理新來的硬體事件，處理完後又會去sleep；RTOS再將控制權交給idle task以進入省電模式。執行時期系統就是依照這樣的循環不停的運作。

 

我們的設計文件中有簡單明瞭的sample code，可以看出main task、idle task與ISR間的互動關係：硬體事件模式予候我們明外域變數統中況的多吧』

 

 

/*************************************************

    Boot 程式

*************************************************/

 

void boot(void)

{

    ...

   

    // 做完硬體與系統初始化後, 才開始建立系統中的task

    // 在RTOS系統中, 基本上一個task就是一個function

    // 觀念如同Windows程式中的thread

    //

   

    // 建立main task, stack size = 2048Byte, 優先等級最高

    //

    rtos_create_task(main_task, STACK_SIZE_2048_Byte, PRIOITY_HIGHEST ) ;

   

    // 建立idle task, stack size = 1024Byte, 優先等級最低

    //

  rtos_create_task(idle_task, STACK_SIZE_1024_Byte, PRIOITY_LOWEST ) ;

   

    // RTOS的排程演算法開始運作, 最高優先等級的task會獲得執行權

    //

    rtos_start() ;

   

    // never go back !

    //

}

 

 

/*************************************************

    Main Task

*************************************************/

void main_task(void)

{

    // 作一些應用程式初始化的動作

    //

    ...

   

    // forever message loop

    //

    while(1)

    {

                    if(get_message(...) == true)

                    {

                                    // process message here

                                    process_message() ;

                                   

                                    // 繼續檢查是否有待處理的message

                                    //

                                    continue ;

                    }

                    else

                    {

                                    // 當目前執行的task進入sleep mode後,RTOS的排程器會起來運作

                                    // 選中優先等級次高的idle task, 使其獲得CPU執行權

                                    //

                                    rtos_task_sleep() ;      

                    }

    }

   

}

 

 

/*************************************************

    Idle Task

*************************************************/

void idle_task(void)

{

 while(1)

 {

           // 讓CPU進入睡眠模式, CPU會暫停執行

           //

           enter_power_saving_mode() ; 

          

           // CPU睡眠模式被硬體事件喚醒, 當ISR執行完畢後

           // idle task會繼續執行

           // 接下來喚醒main task, 使其可以處理新來的硬體事件

           //

           rtos_task_wakeup(MAIN_TASK_ID) ;

 }

   

}

 

 

/*************************************************

    XXX ISR

     - 硬體中斷可以把CPU從睡眠模式喚醒

*************************************************/

void xxx_ISR(void)

{

 // 處理硬體事件

 //      

 ...      

   

 // 將硬體事件傳入main task的message queue中  

 //

 put_message() ;    

           

}
